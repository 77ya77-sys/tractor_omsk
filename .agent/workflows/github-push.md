# Goal
Автоматический коммит и пуш кода в удаленный репозиторий GitHub по команде пользователя ("запушить", "задеплоить").

# Steps
1. Выполнить `git status` для проверки текущего состояния репозитория (инициализировать его через `git init`, если это еще не сделано).
2. Выполнить `git remote -v`.
   - Если удаленный репозиторий (`origin`) отсутствует, прервать выполнение и попросить пользователя прислать URL репозитория.
   - Как только пользователь предоставит URL, выполнить `git remote add origin <URL>`.
3. Выполнить `git add .` для добавления всех изменений в индекс.
4. Проанализировать изменения (через `git diff --cached`) и сгенерировать осмысленное сообщение коммита, описывающее проделанную работу. Выполнить `git commit -m "ваше сообщение"`.
5. Выполнить `git push -u origin main` (или `master`, если текущая ветка называется master).

# Expected artifacts
Синхронизированный проект: локальные изменения зафиксированы и успешно отправлены в удаленный репозиторий на GitHub.
