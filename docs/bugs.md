# Журнал багов проекта

## [Устранено] Блокировка скролла страницы из модального окна в квиз
**Дата обнаружения/исправления:** Февраль 2026

**Описание проблемы:**
При просмотре каталога техники пользователи нажимали карточку (которая открывалась как fullscreen модальное окно вместе с блокировкой прокрутки `document.body.style.overflow = 'hidden'`). В самом модальном окне находилась кнопка (CTA) «Оставить заявку на эту модель», которая вела якорем на блок in-page квиза (`href="#quiz-section"`).
Однако, закрытие модалки было реализовано через inline `onclick` на кнопке, который только убирал визуальный класс (`.is-active`), но не восстанавливал скролл (`document.body.style.overflow = ''`). В итоге пользователь перекидывался к квизу, но прокрутка страницы навсегда оставалась заблокированной (будто невидимый overlay не дает скроллить).

**Решение:**
- В `script.js` при рендере содержимого модалки (функция `openModal`) был изменен inline-код кнопки (класс `modal-detail__cta`). 
- **Код до:** `onclick="document.getElementById('equipment-modal').classList.remove('is-active');"`
- **Код после:** `onclick="document.getElementById('equipment-modal').classList.remove('is-active'); document.body.style.overflow = '';"`

**Результат:**
При клике на CTA-кнопку карточки модальное окно прячется, фокус сбрасывается к квизу, и прокрутка документа (`<body>`) полностью разблокирована, возвращая пользователю контроль над просмотром страницы. Встроенный квиз не накладывает дополнительных блокировок `overflow`.
